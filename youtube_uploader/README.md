# YouTube Uploader for SUNO

Secure, OAuth-based YouTube video uploader. No passwords stored. No browser automation. Fully compliant.

## Setup (5 minutes)

### Step 1: Create Google Cloud Project

1. Go to https://console.cloud.google.com/
2. Create new project: **"SUNO YouTube"**
3. Wait for it to be created

### Step 2: Enable YouTube Data API

1. In Google Cloud Console, go to **APIs & Services**
2. Click **"Enable APIs and Services"** (top button)
3. Search: **"YouTube Data API v3"**
4. Click the result
5. Click **"Enable"** (blue button)

### Step 3: Create OAuth Credentials

1. Go to **APIs & Services** → **Credentials** (left sidebar)
2. Click **"+ Create Credentials"** (top)
3. Choose: **"OAuth client ID"**
4. If prompted: Click **"Configure Consent Screen"** first
   - Choose **"External"**
   - Fill in app name: "SUNO"
   - Add your email
   - Click **"Save and Continue"**
5. Back to Credentials → **"+ Create Credentials"** → **"OAuth client ID"**
6. Choose: **"Desktop application"**
7. Name: **"SUNO Uploader"**
8. Click **"Create"**
9. A popup shows your credentials. Click **"Download"** (JSON)
10. Save file as **`credentials.json`** in this directory

### Step 4: Install Dependencies

```bash
cd youtube_uploader
pip install -r requirements.txt
```

### Step 5: Set Environment Variables

```bash
cp .env.example .env
```

Edit `.env` (optional — only needed if using env vars):
```env
YOUTUBE_CLIENT_ID=your_client_id_here.apps.googleusercontent.com
YOUTUBE_CLIENT_SECRET=your_client_secret_here
```

## Usage

### Basic Upload

```python
from upload_video import YouTubeUploader

uploader = YouTubeUploader()
uploader.authenticate()  # Opens browser for OAuth login (one time)

video_id = uploader.upload_video(
    file_path='my_video.mp4',
    title='My SUNO Clip',
    description='Generated by SUNO AI',
    tags=['suno', 'ai-music'],
    privacy_status='unlisted'
)
```

### Command Line

```bash
python upload_video.py
```

## How it Works

1. **First Run**: Opens browser → You authorize SUNO to upload videos → Token saved locally
2. **Subsequent Runs**: Uses saved token (no browser needed)
3. **Token Refresh**: Auto-refreshes if expired
4. **Security**: Credentials never hardcoded, OAuth token stored locally

## Files

| File | Purpose |
|------|---------|
| `credentials.json` | OAuth credentials (from Google Cloud) — **Keep secret** |
| `token.pickle` | User auth token (auto-created after first login) |
| `upload_video.py` | Main uploader class |
| `requirements.txt` | Python dependencies |
| `.env.example` | Environment variables template |

## Privacy Status Options

- `'public'` — Visible to everyone
- `'unlisted'` — Visible via link only
- `'private'` — Visible only to you

## Troubleshooting

| Error | Solution |
|-------|----------|
| `FileNotFoundError: credentials.json` | Download OAuth credentials from Google Cloud (see Step 3) |
| `Invalid client` | Make sure `credentials.json` is correct |
| `Access Denied` | Grant SUNO permission in OAuth consent screen |
| `File not found` | Check video file path exists |
